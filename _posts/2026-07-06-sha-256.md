# Bitwise Operations
The Secure Hashing Alorithms (SHA) use bitwise operations, it's good to know which ones are used, and how they work.

`AND` operation ($\text{^}$): *returns 1 if both input bits are 1, 0 otherwise*
* `1 ^ 1 = 1`
* `0 ^ 0 = 0`
* `1 ^ 0 = 0`
* `0 ^ 1 = 0`


```
  1010
∧ 0110
------
  0010
```

`NOT` operation ($\text{¬}$): *flips the bit; 1 $\rightarrow$ 0, 0 $\rightarrow$ 1*

```
¬ 101
-----
  010
```

`XOR` operation ($\text{⊕}$): *returns 1 if the two input bits are **not** equal, 0 otherwise*
* `1 ⊕ 1 = 0`
* `0 ⊕ 0 = 0`
* `1 ⊕ 0 = 1`
* `0 ⊕ 1 = 1`


```
  1010
⊕ 0110
------
  1100
```

`SHR` operation (Shift Right): *Slides all bits to the right by $n$ positions. New bits on the left are filled with 0. Bits that fall off the right end are discarded (lost).*

```
# add 3 zeros to the left, remove 3 digits from the right
SHR(3, 10110) = [000] 10 {110} = 00010
```

Python code:

```python
def shr(value, amount):
    """
    See FIPS 180-4, section 3.2.5, for details on the right shift operation.

    SHR(x, n) = x >> n
    """
    return value >> amount
```

`ROTR` operation (Rotatate Right): *Also slides bits right by n positions, but instead of discarding the bits that fall off, they wrap around to the left side.*

```
# move 3 digits from the right to the left side
ROTR(3, 10110) = [110] 10 {110} = 11010
```

Python code:

```python
def rotr(value, amount):
    """
    See FIPS 180-4, section 3.2.4, for details on the right rotation operation.

    ROTR(x, n) = (x >> n) v (x << w - n)
    Where
        w   =   the word size (32 bits for SHA-256)
        v   =   is the bitwise OR operation.

    0xFFFFFFFF is a 32-bit value of all 1's. Using the bitwise AND operation
    with 0xFFFFFFFF ensures that the result is a 32-bit value, effectively
    discarding any bits beyond the 32nd bit.
    """
    return (value >> amount | value << (32 - amount)) & 0xFFFFFFFF
```


# SHA-256 Functions
The SHA-256 Algorithm relies on 6 logical functions, all of which take as input 32 bit "words" and output 32 bit "words" (in this context a "word" is a binary encoding of a string).

**Choose function:**
$$\text{Ch}(x, y, z) = (x \land y) \oplus (\neg x \land z)$$

**Majority function:**
$$\text{Maj}(x, y, z) = (x \land y) \oplus (x \land z) \oplus (y \land z)$$

**Sigma 0:**
$$\Sigma_0(x) = \text{ROTR}(2, x) \oplus \text{ROTR}(13, x) \oplus \text{ROTR}(22, x)$$

**Sigma 1:**
$$\Sigma_1(x) = \text{ROTR}(6, x) \oplus \text{ROTR}(11, x) \oplus \text{ROTR}(25, x)$$

**sigma 0:**
$$\sigma_0(x) = \text{ROTR}(7, x) \oplus \text{ROTR}(18, x) \oplus \text{SHR}(3, x)$$

**sigma 1:**
$$\sigma_1(x) = \text{ROTR}(17, x) \oplus \text{ROTR}(19, x) \oplus \text{SHR}(10, x)$$

```python
choose = lambda x, y, z: (x & y) ^ (~x & z)
majority = lambda x, y, z: (x & y) ^ (x & z) ^ (y & z)

Sigma0 = lambda x: rotr(x, 2) ^ rotr(x, 13) ^ rotr(x, 22)
Sigma1 = lambda x: rotr(x, 6) ^ rotr(x, 11) ^ rotr(x, 25)
sigma0 = lambda x: rotr(x, 7) ^ rotr(x, 18) ^ shr(x, 3)
sigma1 = lambda x: rotr(x, 17) ^ rotr(x, 19) ^ shr(x, 10)
```
