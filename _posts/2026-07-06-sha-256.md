The SHA-256 hashing algorithm is widely used in the fields of cryptography, cyber security, and back-end web development. Although it's not necessary to understand how this algorithm works under the hood in most situations, I do find it interesting. This article takes a deep dive in the internal workings of the algorithm, and how you can implement it yourself.

This article is written based on the official *Secure Hashing Standard Specification* (see Appendix C). For the implementation of the algorith in Python from scratch, see Appendix B.

# 1 Pre-Processing
SHA-256 expects as initial input (message $M$) as a series of $N$ number of 512-bit blocks. Each block $B_i$ is a bit-string (a sequence of bits: 1's and 0's) of length 512. How these blocks are created is according to a set of pre-processing instructions.

$$
M = B_1, B_2, ... , B_N
$$

Each block $B_i$ is broken up into 16 "words", where each word $w_j$ is a bit-string of length 32.

$$
B_i = w_1, w_2, ... , w_{16}
$$

> **Note**: *I am deviating from the notation in the specification, as they refer to the message in its entirety, the message blocks, and the "words" by the same variable $M$ but with various subscripts and superscripts. I find this can lead to some confusion, so I've introduced my own variable names*

The official specifications work in a very generalized manner, only looking at bits. However, as we are dealing with ASCII or utf-8 encoded text, and because the algorithm was designed with text input in mind, it is easier to work with bytes (chunks of 8 bits).
* Message $M$ consists of $N$ blocks $B$
* each block $B$ consists of 16 words $w$
* each word $w$ consists of 4 bytes
* 1 byte = 8 bits

| Entity | Symbol | Child-Entity | Number of Children | Size in bits |
| - | - | - | - |
| Message | $M$ | Block | $N$ | 512 $\times N$ |
| Block | $B$ | Word | 16 | 512 |
| Word | $w$ | bytes | 4 | 32 |
| byte | [-] | [-] | [-] | 8 |

The pre-processing step consists of 3 steps:
1) encoding the raw string
2) padding the resulting bits
3) dividing the message into blocks & words

### Step 1) Encoding
We will use either ASCII or utf-8 encoding to transform each character in the input string `"abc"` into a series of bytes.

```
   a        b        c
01100001 01100010 01100011
```

### Step 2) Padding
The next step is to pad the message in a very specific way. First a seperator byte `10000000` is added to the message.

```
   a        b        c        seperator
01100001 01100010 01100011    10000000
```

The last 8 bytes are reserved for the the length $l_b$ (in bits) of the original message. In this case the original message is 3 characters, and the encoding assigns 1 byte to each chracter, so the length of the original message is given by $l_b = 3 * 8 = 24$, which in binary is `11000`

```
   a        b        c        seperator      length
01100001 01100010 01100011    10000000    {00 ... 0011000}
```

Finally, the message in total needs to be a multiple of 512 bits, or in other words the message in total needs to be a multiple of 64 bits, so the rest of the message needs to be padded with empty bytes `00000000`. In general the number of empty padding bytes can be found by solving for $k$ in the following expression.

$$(k + l + 1) \mod 64 = 56$$

$l + 1$ is the integer value of the length of the input string $l$ (3 in our case), +1 for the seperator byte. Each block must be 64 bytes, and of those 64 blocks 8 are reserved to describe the length, hence the number 56 is used on the RHS of the equation.

In our case we have 3 bytes for the original message, 1 seperator byte, and 8 bytes to describe the length, which means we need 64-3-1-8 = 52 empty bytes. These empty bytes are always placed between the seperator and the length bytes.

```
   a        b        c        seperator       padding         length
01100001 01100010 01100011    10000000      {00 ... 00}     {00 ... 0011000}
```

### Step 3) Grouping
At this point we have a large sequence of bytes. Due to how the number of empty padding bytes was determined, we know the total length of the message $M$ (in bytes) at this point is a multiple of 64, we can very easily group the message into blocks $B_i$ of length 64.

In our example the input string `"abc"` is short enough that everything fits into a single block with a bunch of empty bytes.

```
blocks[0] =
  01100001 01100010 01100011 10000000 00000000 00000000 00000000 00000000
  00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
  00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
  00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
  00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
  00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
  00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
  00000000 00000000 00000000 00000000 00000000 00000000 00000000 00011000
```

Each group is then further split into a set of words. According to the specification, each word is a 32-bit integer, which is equivalent to 4-bytes

```
blocks[0] = [w[0], w[1], ... , w[15]]

w[0] = 01100001 01100010 01100011 10000000
w[1] = 00000000 00000000 00000000 00000000
...
w[14] = 00000000 00000000 00000000 00000000
w[15] = 00000000 00000000 00000000 00011000
```

> NOTE: If we are implementing this in Python, the computation & arithmatic works a lot nicer with integers, in which case words will be cast to big-endian 32-bit sized decimal integers.
>
>```w[0] = 01100001 01100010 01100011 10000000 = 1,633,837,952```

# 2 Hashing Algorithm
This section covers how the the prepared (pre-processed) message is transformed in order to obtain a hash string from it.

## 2.1 Preparation
Here is an overview of all of the constants and functions that will be used in the Hashing Computation.

### 2.1.2 Initial Hash Values
The SHA-256 algorithm starts with an intial set of values: the first thirty-two bits of the fractional parts of the square
roots of the first eight prime numbers

```
h0 := 0x6a09e667
h1 := 0xbb67ae85
h2 := 0x3c6ef372
h3 := 0xa54ff53a
h4 := 0x510e527f
h5 := 0x9b05688c
h6 := 0x1f83d9ab
h7 := 0x5be0cd19
```

### 2.1.3 Functions
The SHA-256 Algorithm relies on 6 logical functions, all of which take as input 32 bit "words" and output 32 bit "words" (in this context a "word" is a binary encoding of a string).

**Choose function:**
$$\text{Ch}(x, y, z) = (x \land y) \oplus (\neg x \land z)$$

**Majority function:**
$$\text{Maj}(x, y, z) = (x \land y) \oplus (x \land z) \oplus (y \land z)$$

**Sigma 0:**
$$\Sigma_0(x) = \text{ROTR}(2, x) \oplus \text{ROTR}(13, x) \oplus \text{ROTR}(22, x)$$

**Sigma 1:**
$$\Sigma_1(x) = \text{ROTR}(6, x) \oplus \text{ROTR}(11, x) \oplus \text{ROTR}(25, x)$$

**sigma 0:**
$$\sigma_0(x) = \text{ROTR}(7, x) \oplus \text{ROTR}(18, x) \oplus \text{SHR}(3, x)$$

**sigma 1:**
$$\sigma_1(x) = \text{ROTR}(17, x) \oplus \text{ROTR}(19, x) \oplus \text{SHR}(10, x)$$

> **Note**: *See Appendix A for more information about bitwise opertions used in these functions, and how they can be implemented in python. Also notice how python and the specification document's notation are different for these operations*

```python
choose = lambda x, y, z: (x & y) ^ (~x & z)
majority = lambda x, y, z: (x & y) ^ (x & z) ^ (y & z)

Sigma0 = lambda x: rotr(x, 2) ^ rotr(x, 13) ^ rotr(x, 22)
Sigma1 = lambda x: rotr(x, 6) ^ rotr(x, 11) ^ rotr(x, 25)
sigma0 = lambda x: rotr(x, 7) ^ rotr(x, 18) ^ shr(x, 3)
sigma1 = lambda x: rotr(x, 17) ^ rotr(x, 19) ^ shr(x, 10)
```

## 2.2 Hash Computation
The algorithm itself is relatively simple. The sections below describe the general flow and logic of the algorithm. The code snippets show pseudo-python-code to illustrate the ideas/concepts. For the actual python code required to implement this, see Appendix B.

![flow]({{ site.baseurl }}/assets/images/sha256/sha256_algorithm_flow.png)

### Step 1) Initialize Initial Hash Values
We simple set the initial values of $H_0$ through $H_7$ equal to those defined in the standard (see section 2.1.2 of this blog post).

```python
INITIAL_HASH = [H0, H1, H2, H3, H4, H5, H6, H7]
```

### Step 2) Loop Over Blocks
For each block will will perform a set of computations. The block itself is a list of 16 "words".

#### Step 2.1) Prepare Messaging Schedule
We will first prepare a "messaging schedule" `w`, which is nothing more than an array of 64 values. The first 16 values are equal to the 16 "words" ($w_j$) that make up the current block (block is a list of words). The remaining values in the array (indices 16-63) are computed on the fly using the `sigma1` and `sigma0` functions.

```python
# 0 <= t < 16
w[t] = block[t]

# 16 <= t < 64
w[t] = sigma1(w[t-2]) + w[t-7] + sigma0(w[t-15]) + w[t-16]
```

#### Step 2.2) Assign Current Hash Values to Temporary Variables
First we will assign each of the current values of $H_0$ through $H_7$ to a temporary variable $a$ through $h$

```python
a,b,c,d,e,f,g,h = H0,H1,H2,H3,H4,H5,H6,H7
```

#### Step 2.3) Transform Values of Temporary Variables
The main computation is in this step, where we assign new values to the temporary variables using their current values. We loop over all 64 values of the "messaging schedule", and update the values of $a$ through $h$ each iteration using the following expressions:

```python
T1 = (h + Sigma1(e) + choose(e, f, g) + K[t] + w[t])
T2 = (Sigma0(a) + majority(a, b, c))
h = g
g = f
f = e
e = (d + T1)
d = c
c = b
b = a
a = (T1 + T2)
```

#### Step 2.4) Update the Current Hash Values using Temporary Variables
When we have finished looping over every value in the messaging schedule, we update the hash values $H_0$ through $H_7$ using the newly computed values of $a$ through $h$
```python
H0 += a
H1 += b
H2 += c
H3 += d
H4 += e
H5 += f
H6 += g
H7 += h
```

### Step 3) Concatenate Hash Values
Finally, after we have updated the hashing values by running the aforementioned transformations for each block, we have our final set of hash values. These hash values are then concatenated, and often returned as a hex string.

```python
hash = to_hex(H0 || H1 || H2 || H3 || H4 || H5 || H6 || H7)
```
